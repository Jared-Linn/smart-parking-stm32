# 智能停车场管理系统 — 设计文档

> **项目名称**：基于 STM32F103 的智能停车场管理系统  
> **硬件平台**：移动互联开发平台 STM32 V2.0（百科荣创）  
> **开发环境**：Keil MDK 5.25 + STM32F1xx DFP 2.1.0  
> **设计日期**：2026.07.04

---

## 一、项目背景

随着城市车辆保有量持续增长，传统停车场人工收费、人工引导的方式效率低下。本项目模拟一个**智能化社区停车场管理系统**：车辆进场自动检测、RFID 刷卡抬杆、RTC 计时计费、环境感知自动照明、WiFi 云端远程监控，实现停车场的无人化管理。

---

## 二、系统功能

### 2.1 核心功能

| 编号 | 功能 | 描述 |
|------|------|------|
| F1 | 车辆进出检测 | HC-SR04 超声波检测车位前是否有车辆 |
| F2 | RFID 刷卡 | MFRC522 13.56MHz 读卡，支持 MIFARE Classic 1K |
| F3 | 道闸控制 | 舵机 0°~90° 模拟道闸抬杆/落杆 |
| F4 | 计时计费 | RTC 记录进出时间，按停车时长阶梯计费 |
| F5 | 环境监测 | BH1750 光照度 + MLX90614 温度 |
| F6 | 智能照明 | 光照不足时自动开启 WS2812 照明 |
| F7 | 信息显示 | 0.96" OLED 显示车位状态、时间、费用 |
| F8 | 云端监控 | ESP8266 WiFi → MQTT Broker → 远程状态上报 + 命令接收 |
| F9 | 串口 CLI | USART1 命令行接口，支持 PC 端控制 |
| F10 | 系统保护 | IWDG 看门狗，1.5 秒喂狗防死机 |

### 2.2 扩展功能

| 编号 | 功能 | 描述 |
|------|------|------|
| E1 | 内部温度对比 | ADC1 CH16 采集 MCU 内部温度，与 MLX90614 环境温度对比 |
| E2 | DAC 电压输出 | DAC CH1 (PA4) 输出模拟电压，模拟外部传感器信号 |
| E3 | DMA 串口 | DMA1 CH4 传输 USART1 TX，大数据量上报不占 CPU |

### 3.2 完整引脚分配

| 引脚 | 功能 | 方向 | 备注 |
|------|------|------|------|
| PA0 | TIM2_CH1 / ADC1_CH0 | 复用 | 舵机 PWM (50Hz) |
| PA1 | ADC1_CH1 | 输入 | 备用 ADC |
| PA2 | TRIG / SCL | 输出 | 超声波触发 / BH1750 SCL (分时复用) |
| PA3 | ECHO / SDA | 输入 | 超声波回波 / BH1750 SDA (分时复用) |
| PA4 | RC522 MISO / DAC1 | 输入/输出 | SPI MISO / DAC 输出 |
| PA5 | RC522 MOSI | 输出 | SPI MOSI |
| PA6 | RC522 SCK | 输出 | SPI SCK |
| PA7 | RC522 NSS | 输出 | SPI 片选 |
| PA8 | BEEP (TIM1_CH1) | 输出 | 蜂鸣器 PWM |
| PA9 | USART1 TX | 输出 | 串口1 发送 |
| PA10 | USART1 RX | 输入 | 串口1 接收 |
| PB8 | OLED D1 (MOSI) | 输出 | OLED SPI 数据 |
| PB9 | OLED D0 (SCK) | 输出 | OLED SPI 时钟 |
| PB10 | USART3 TX | 输出 | ESP8266 发送 |
| PB11 | USART3 RX | 输入 | ESP8266 接收 |
| PD8 | LED0 | 输出 | 车位占用指示 |
| PD9 | LED1 | 输出 | 系统心跳 |
| PD10 | LED2 | 输出 | WiFi 状态 |
| PD11 | LED3 | 输出 | 报警指示 |
| PE1 | OLED DC | 输出 | OLED 数据/命令 |
| PE2 | OLED RST | 输出 | OLED 复位 |
| PE3 | OLED CS | 输出 | OLED 片选 |
| PE7 | KEY1 | 输入 | 进场请求 |
| PE8 | KEY2 | 输入 | 出场请求 |
| PE9 | KEY3 | 输入 | 查询记录 |
| PE10 | KEY4 | 输入 | 管理员模式 |

> **注意**：BH1750/MLX90614 共用 PA2(SCL)/PA3(SDA) 软 I2C 总线。超声波也使用 PA2(TRIG)/PA3(ECHO)。这三个模块不能同时连接，通过扩展口 P3 切换使用。

### 4.4 关键数据结构

```c
// 停车场状态枚举
typedef enum {
    PARKING_IDLE = 0,       // 空闲
    PARKING_WAIT_CARD,      // 等待刷卡
    PARKING_GATE_OPEN,      // 道闸打开中
    PARKING_GATE_CLOSE,     // 道闸关闭中
    PARKING_PARKED,         // 已停车
    PARKING_EXIT_WAIT,      // 等待出场刷卡
    PARKING_EXIT_OPEN,      // 出场抬杆
    PARKING_ALARM           // 报警状态
} ParkingState;

// 停车记录
typedef struct {
    uint8_t  card_id[4];    // 卡号
    uint32_t entry_time;    // 入场时间 (RTC计数)
    uint32_t exit_time;     // 出场时间
    uint16_t fee;           // 费用 (分)
    uint8_t  valid;         // 记录是否有效
} ParkingRecord;

// 系统全局状态
typedef struct {
    ParkingState state;         // 当前状态
    ParkingRecord current;      // 当前停车记录
    uint8_t  card_detected;     // 检测到卡片
    uint8_t  car_present;       // 车辆在位
    uint16_t distance_cm;       // 超声波距离
    uint16_t light_lx;          // 光照度
    float    temperature;       // 温度
    uint8_t  wifi_connected;    // WiFi状态
    uint8_t  alarm_flag;        // 报警标志
} ParkingSystem;
```

## 五、通信协议

### 5.1 MQTT 云端协议

**上报 Topic**: `parking/{device_id}/status`

```json
{
  "device_id": "PARK001",
  "state": 1,
  "occupied": true,
  "temperature": 28.5,
  "light_lx": 320,
  "time": "2026-07-04 14:30:25"
}
```

**命令 Topic**: `parking/{device_id}/cmd`

```json
{"cmd": "open_gate"}
{"cmd": "light_on"}
{"cmd": "light_off"}
{"cmd": "report"}
```

### 5.2 串口 CLI 协议

| 命令 | 功能 | 响应 |
|------|------|------|
| `status` | 查询系统状态 | 返回 JSON 状态 |
| `open` | 远程开闸 | `OK: Gate opened` |
| `close` | 远程关闸 | `OK: Gate closed` |
| `log` | 打印最近记录 | 列表格式 |
| `reset` | 系统复位 | `OK: Resetting...` |
| `help` | 帮助信息 | 命令列表 |

### 5.3 RFID 卡数据格式

使用 MIFARE Classic 1K 卡，Block 4 存储余额数据：

| 字节 | 0-1 | 2 | 3-15 |
|------|-----|---|------|
| 含义 | 卡号 | 余额(元) | 保留 |
